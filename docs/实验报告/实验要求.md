1. 实践目的

本次实践任务主要目标为运用数据库课程的基础理论知识解决现实世界中的具体问题。能够设计并实现一套满足相关功能需求、设计合理、安全、稳定、简单易用的系统。其中，数据库相关知识包括 SQL、存储过程、存储函数、触发器等。旨在培养学生知识运用能力、资料搜索能力、团队协调能力、文档撰写能力等在未来工作学习岗位中所需具备的基本能力。

2.实践形式

本次实践具体要求如下：
1) 本次实践不分组，独立完成；
2) 本次实践为一道题，题目中数据库表等内容自定定义；
3) 每位同学需按时完成所分配任务，并进行项目答辩。包括：
a)   答辩前每个小组需将所有项目资料在 Git  系统中上传，现场答辩评分将参考所提交资料；
b)   准备好 PPT 文件及可访问的系统；
c)   PPT 文件包括实验报告中所需包含的内容（参考第 6 条）等；
d)   报告总时间不超过 10 分钟；
e)   报告人使用自己笔记本电脑，通过腾讯会议形式进行报告。提前准备好软硬件环境；
f)    在完成报告后进行现场答辩。
4) 每位同学提交实验报告，实验报告内容包括但不限于需求分析、系统架构设计、数据库设计、数据库技术应用方法、运行实例等内容；
5) 最终评分包括：考勤分（10%）、报告分（20%）、代码评分（20%）、答辩评分（50%）。

3.实践要求

本次实践作品需满足以下要求：
1) PC 端：界面美观合理、兼容 Chrome、Firefox、Edge 等主流浏览器、能够适应不同分辨率、操作便捷、包含对第三方接口调用、提供数据可视化；
2) 服务端：提供前端访问接口。系统需部署在 docker 环境中；
3) 数据库方面：

a)    能依据具体业务需求，分析数据实体及其关系，准确绘制 E-R   图，识别实体、属性、联系、主键与外键；能将 E-R 模型转化为合理的关系模式，并通过 1NF～3NF（甚至 BCNF）等规范化过程提高数据结构的合理性与可维护性。
b)   能根据系统访问特征与性能要求，合理设计索引结构、数据分区方案，选择合适的存储引擎，并对查询性能进行分析与优化，实现数据库的高效运行。
c)    能熟练使用 SQL 完成数据库对象的定义与操作，包括表的创建、插入、更新、删除、约束与视图的管理；能够编写涉及多表连接、聚合、嵌套子查询等复杂 SQL 语句，解决实际业务查询问题。
d)   能设计并实现完整的事务逻辑，正确处理并发与异常情况，掌握事务的提交、回滚与一致性维护，确保数据安全可靠。
e)    能进行数据库用户的创建与管理，合理分配访问权限，实施数据加密与审计策略，保障系统安全与合规性。
f)    能编写数据导入、导出与备份脚本，掌握数据库日常维护、迁移与恢复操作，提升系统可用性与数据可持续性。
g)   能在数据库端实现存储过程、触发器与游标等程序对象，完成复杂业务逻辑处理与数据自动化操作，体现数据库系统编程与扩展的综合工程能力。
4)   Git：所有的源代码、数据库备份文件、实验报告、 PPT、演示视频等都需要通过 git 方式进行提供（详见附录 Git 使用说明书）；开发过程中需使用 git 进行版本管理；
5) 所写代码需符合代码规范。

4.实践任务

该数据库同步系统为对多数据库进行数据实时和在线同步。具体需求包括：
1)    需包含不少于三种数据库系统（如 MySQL 、Oracle  等）（支持表迁移，整库迁移）；
2)   所有数据库中表结构一致；
3)    不少于 5 张表（数据库表内容自行定义）；
4)   包含用户管理及权限管理；
5)   提供一个页面，对数据库中某些数据进行查询，其 SQL 语句包含多表连接、嵌套子查询等，同时，数据库段能对查询效率进行优化。

6)    同步操作为保持各表中数据一致；
7)   系统后端能够支持实时、定时两种同步方式。实时为某一数据库发生变化即立刻同步到其他数据库。定时为系统根据配置定时通过周期和时间进行自动同步；
8)   对于同步中数据冲突情况，能及时通过邮件方式提示管理员；
9)   管理员点击邮件或者短信中链接可以在在移动端和 PC 端能够查看异常数据冲突情况。链接中包含身份认证信息。移动端和 PC 端能通过链接中参数进行身份验证；
10) 管理员能在 PC 端对冲突数据进行处理，对于每条记录选择以哪个数据库数据为准进行同步；
11) 管理员在移动端可以查看每天数据同步情况的分析报表（图形方式）*；
